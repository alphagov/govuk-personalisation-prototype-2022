Round 1

Tasks

  1. Interact with Get Childcare step-by-step
  2. Create a GOV.UK account
  3. Interact with HMRC Childcare Calculator
  4. Consent to save data inputted in HMRC Chldcare Calculator to GOV.UK Account
  5. [possible] interact with OFSTED ‘find childcare’ page and concept check use of postcode from saved data
  6. Apply for Universal Credit
  7. Consent to use stored data from GOV UK account in the Universal Credit application
  8. Consent to share additional data which was inputted as part of the Universal Credit application
  9. Forced log out and log in
  10. HMPO renew passport
  11. Consent to use stored data from GOV UK Account in HMPO renew passport application

Goals

Goal 1

Explore user perception of Gov Account creation within the context of a x-gov service. i.e. perceived pros and cons

Goal 2

Explore when, how, and how often to ask for consent for data sharing

Goal 3

Explore where to position the Gov Account within the user journey

Goal 4

Explore UI options throughout e.g. pre-population, progress indicator

Goal 5

Elicit user reaction to saved data for future use


Assumptions

  1. All services are connected via broker API
  2. All services use the GOV.UK Account SSO
  3. User needs to be signed in to GOV.UK Account for data/activity to be captured
  4. Save and return is available on all forms
  5. Legacy data requires user to PYI (out of scope for MVP)
  6. This is a future view where service would already be consumed by GOV.UK account. Therefore this research does not cover account confusion and issues around onboarding.

Hypotheses

Hypothesis 1

We believe that conveying the benefits of a Gov Account within a x-gov transactional journey
For users of Get Childcare
Will increase willingness to sign up for a Gov Account

Hypothesis 2

We believe stating where the data will be shared and how the data will be used
For users of Get Childcare
Will increase willingness to grant consent

Hypothesis 3

We believe pre-filling forms
Will speed up task completion
We will know this if / when user research participants express potential benefits

Hypothesis 4

We believe pre-populating forms
Will be perceived as a valuable feature of the gov account
We will know this if / when user researcher participants cite this as a potential benefit

Hypothesis 5

We believe offering greater transparency and control over data
Will enhance trust and confidence
We will know this if / when user research participants express interest in ways to interact with and control their data items

Hypothesis 6

We believe stating where the data will be shared and how the data will be used
Will increase willingness and confidence in saving and using data for future transactions
We will know this if / when users react positively to their data being used in an unrelated service (see task list)
