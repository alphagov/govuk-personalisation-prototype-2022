Users perceived that the use of the ‘verified’ tag indicated that the information had been checked. Some users assumed that verification related to their identity. Some users perceived ‘verified’ as a way of indicating task completion or progress.

Examples from user research:

“They’re checking it’s me and this is my information.” from phase 2, round 3

“They’ve either checked it against the ID that’s been uploaded or done some sort of search where they can verify ID in that way.” from phase 2, round 2

“They’ve checked that it’s been completed, that everything’s fine and everything checks out.” from phase 2, round 2

Users expected that ‘verified’ or ‘trusted’ data would enable quicker future transactions as that data would be accurate

Examples from user research:

“It would make things a lot easier, it’s all there ready and there’s nothing you need to add in or out back in yourself. It’s easier.” from phase 2, round 2

“Once it’s verified it’s done and you don’t have to think about it.” from phase 2, round 4
