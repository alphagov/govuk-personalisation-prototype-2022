• Users already expect that government is joined up and departments talk/exchange data with one another.
  • It has to be an explicit and meaningful interaction to use global push, for example 'tick to tell HMRC XYZ about me'.
  • Global push would reduce the need for services to send reminders to users to update their details. Some reminders can cause a lot of frustration and upset (depending on the sensitivity of the service).
  • It could be used for informing services about potentially fraudulent accounts.
  • Hygiene checks would be needed. For example, would the user need to renew their global push permissions every 90 days? Consent needs to prompt a hygiene check "who is seeing my data?"

If we knew that data is reliable then we could communicate with customers better, being safe in the knowledge that that is up to date, as reliable as needed it to be...a better form of communication.
