Services felt that this would be valuable - but with the caveat that people will be either really pleased by it or mortified to know how much information some departments have about them.

There will be elements of existing data that would be really useful to be shared across services, but many services acknowledged that there will be considerable amounts of data that will only ever be specific to a given service. There is a risk that over time, the user will build up a huge profile/data repository the more government services they use.

There may be some things that help us identify people but there will be some things that are only specific to the service. It will become richer when people have done several applications over time - and the service builds up confidence in the person and it's easier for the user. [DEFRA]

Service use case

  • "Identity is one thing we have to check. If it's already checked in the existing data that would save us our £1.10 check that we have to do. We're all (All Local Authorities) forking out that amount for something that is already in our grasp." [reference to local authorities using an external address checker which costs £1.10 per check] West Northamptonshire Council - Blue Badge
  • [Valuable to get existing verified payment details] "Bank details and IDV if the user has already gone through the process, we could flag up 'is this still your bank details'... this would speed up payablilty aspect." DWP
