As a future consideration, any management controls will need to tolerate the potential for data rich accounts. There will likely be a future design challenge around how to display, group and organise larger data sets.

What data can be maintained:

A user can access and maintain unverified data - but they should not be able to maintain "verified credentials".

Alter verified data:

If a user wants to alter/update verified data - how do they do this? How do they know which services they need to go back through to alter this?

Change of "key" verified data:

If a user alters a key verified data element (such as DOB), should this cause *all* verified credentials to become invalid?
