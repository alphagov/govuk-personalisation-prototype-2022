Considerations:

  • Fixing data gives the impression that personal data does not belong to the person
  • Users perception of global push and how we would manage their expectations/understanding of how it works
  • You would need a mechanism/process for the user to change fixed data if it's incorrect
  • If data is fixed and it's incorrect, it could delay users from updating something/accessing a service
  • Data would need to be verified and locked until a certain date. What happens at expiry/when this date is reached?

Risk:

Would need to be careful that data designated as fixed may change in future "Be careful in future as things like gender may not be fixed"

Data:

Services assumed that in order for data to become fixed it would have to be verified first.

Operating model

There was concern from services about the operating model - if the data is wrong somewhere, you have to be able to change it - what is the support mechanism? If the user contacts the contact centre - who's contact centre is this? Will the GOV.UK GDS contact centre need to resolve this or the service the data has come from? How will this be explained to the user?
