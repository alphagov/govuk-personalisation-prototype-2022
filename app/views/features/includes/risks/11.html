Audit log of deleted data:

A challenge may be presented in the link between the audit log and 'delete data'.

For example, would an audit log allow users to view deleted items and how they had previously been used? Or, would 'delete' remove the record entirely?

These concepts have not been tested with users so we would recommend further scenario mapping and user testing here.

Contact details / PDS:

If a user deletes their contact details and a service *only* uses the PDS, then the service has no way to contact the user. Then what happens?

Data restore:

If this is a personal data store, there is no responsibility for anyone to backup this data. If it is corrupted or deleted - what happens?

Is there a temporary rollback? Whose responsibility is it to do this? (for example, any data stored on a user's phone is *their* responsibility to backup - is this the same with the PDS?)

Does delete mean delete:

If a user deletes their data on their PDS, this data will still exist in a service. How could/should this change when 08. Global push becomes a possibility. Should services rely on their own data outside of a transaction?
