Data needs to be of good quality, therefore data should be verified before a global push takes place

Policy considerations:

  • They suggest that users should be given the choice about whether or not their data gets pushed to other services
  • There needs to be transparency about how global push works, what it means and how their data is going to be used is key
  • Public concern about global push becoming a 'surveillance tool' for government
  • Accuracy is crucial, consideration needs to be taken where users have multiple pieces of the same data (for example email addresses, addresses) - the user taking control of their data is crucial

Tech risks / challenges / considerations:

There is a team (GDX?) working on the Pub/Sub model - this will fundamentally affect the execution of this, or a similar feature.
