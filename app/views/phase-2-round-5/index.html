Phase 2 / Round 5

To walkthrough this prototype, follow the tasks below (suggestion: keep them open on a separate tab).


Tasks

  1. Start on GOV.UK homepage Â 
  2. Review GOV.UK account explained
  3. Sign in and review signed in GOV.UK homepage
  4. Using top link in the black box, review Coronavirus information page (not personalised at this point)
  5. Go to account and review all 4 account pages
  6. Turn on Reminders feature on account features
  7. Return to account dashboard to review an example reminder
  8. Turn on both personalisation features on account features
    • The first personalisation toggle will hide content deemed not relevant
    • The second toggle will personalise any dates, amounts or locations
  9. Return to GOV.UK homepage

    This page is personalised in two ways:
    • Lots of content and links have been stripped out, leaving those which the user uses most frequently.
    • If the second toggle is on, the Coronavirus links and Bank Holidays panels will be personalised to the users home address.
  10. Go back to Coronavirus page

    This page is now personalised in two ways:
    • Irrelevant content (such as 'Self-employment guidance', and school-based advice in 'Childcare and parenting') is hidden.
    • Personalised locations are used for booking tests, vaccinations, help and guidance, and statistics.
  11. Return to GOV.UK homepage
  12. Go to Child Benefit Payment date

    This page is also personalised in two ways:
    • Irrelevant content (such as 'first payment') is hidden.
    • Personalised dates and amounts are shown in the personalised panel
    • Unhappy path content ("If you havenâ€™t got your payment") is always shown
  13. Go back to account features and turn on Suggestions feature
  14. View suggestions on Account dashboard (they are also shown in the black box on the GOV.UK homepage)
  15. Also, the previously visited Child Benefit Payment date now includes the suggestion to add second child to the claim
  16. Sign out

Draft

  1. Understand whether users are willing to use their data for personalisation.
  2. Understand perceived value of different types of personalised features and content i.e. recommendations for information, recommendations around eligibility.
