Phase 2 / Round 1

Tasks

  1. Start on GOV.UK homepage
  2. View GOV.UK account information page
  3. Go to Childcare step-by-step
  4. Select Child Benefit claim
  5. Sign in to their GOV.UK account
  6. Complete Child Benefit claim
  7. Consent to save inputted data at individual data item level
  8. View account area
  9. Sign out..... time passes
  10. Start on GOV.UK homepage
  11. Sign in to their GOV.UK account
  12. Review account area:
    1. Global controls
    2. Activity log
    3. Data items (trusted/verified)
    4. Reminders
  13. Go to Childcare step-by-step
  14. Start UC application
  15. Consent to use saved data from GOV.UK account (inc. trusted data item)
  16. Add additional information in UC application
  17. UC application summary page
  18. Consent to save inputted data at individual data item level
  19. Go to Childcare step-by-step
  20. Review curated content information pages (Tax-free childcare) with personalisation toggles

Objectives

  • Explore user understanding of the role of the GOV.UK account within x-gov transactions (continued from phase 1).
  • Identify whether any confusion persists around saving data to GOV.UK account vs. using data in a service (continued from phase 1).
  • Explore user understanding of the concept of trusted data, and gather their expectations around what this could enable.
  • Explore whether user is willing (hypothetically) to use their GOV.UK account data to personalise content, and explore to what extent content personalisation is comfortable and appropriate.
  • Identify whether confusion persists around information used to personalise content vs. information used on a service results page (continued from phase 1).
  • Find out whether there is an expectation that global push will occur, and if so, what creates that expectation.
  • Explore user understanding of profile / dashboard features, and gather their expectations in relation to what these features could do, or could enable.

Hypotheses

Hypothesis 1

We believe that more explicit content on consent pages will enable better informed consent to save and use data within the GOV.UK account.

We acknowledge that in real life, not all users will read the content and some confustion may still persist - in reference to our user research goals, we will continue to explore this.

Hypothesis 2

We believe that global consent management will serve users with greater transparency and control and meet the user need of feeling safe and secure in the service. We will know this if users react positively to this feature in user testing.

Hypothesis 3

We believe that an activity log will remove the need to remember transactions and manage paperwork, meeting the user need of reducing cognitive load. We will know this if users react positively to this feature in user testing and appraise the feature in a way which references this user need.

Hypothesis 4

We believe that editable / non-editable / verified data will be understood as something which is fixed and certified by a service

Hypothesis 5

We believe that hiding irrelevant content as part of content curation will be received positively by users as long as they can control this in settings
