Goals

Goal 1

Explore user perception of Gov Account creation within the context of a x-gov service. i.e. perceived pros and cons

Goal 2

Explore extent to which content can enable understanding of Gov Account

Goal 3

Test scenario to resolve around data being used in service and saved to GOV acount (see r1 and r2 insights)

Goal 4

Elicit user reaction to personalisation and curated content

Goal 5

Elicit user reaction to full auto-complete (see renew passport task)


Assumptions

  1. All services are connected via broker API
  2. All services use the GOV.UK Account SSO
  3. User needs to be signed in to GOV.UK Account for data/activity to be captured
  4. Save and return is available on all forms
  5. Legacy data requires user to PYI (out of scope for MVP)
  6. This is a future view where service would already be consumed by GOV.UK account. Therefore this research does not cover account confusion and issues around onboarding.

Hypotheses

Hypothesis 1

Hypothesis 2

We believe more explicit content on consent pages
Will enable a more informed consent to save and use data within the GOV.UK account
We will know this when user research participants can confidently explain that their data is being used by the service and saved to the gov account

Hypothesis 3

We believe offering save and share consent over individual data items
Will enhance trust in gov accounts
We will know this if / when user research participants cite this as a positive feature

Hypothesis 4

We believe full pre-population / auto-complete of a transactional form
Will save users time
We will know this if / when user research participants cite this as a perceived benefit

Hypothesis 5

We believe full pre-population / auto-complete of a transactional form
Will be a valuable and trusted feature
We will know this if / when user research participants can confidently explain why data has passed from one gov service to another via the GOV.UK account [see renew passport task]
