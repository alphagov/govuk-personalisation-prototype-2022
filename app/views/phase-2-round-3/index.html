Phase 2 / Round 3

To walkthrough this prototype, follow the tasks below (suggestion: keep them open on a separate tab).


Tasks

  1. Start on GOV.UK homepage
  2. Nav to account - participant reviews account explainer
  3. Sign in
  4. Account area - add personal details
  5. Prove your identity
  6. Explore ‘verified’ tags
  7. Request saved information - legacy data request
  8. Select services
  9. Select all information
  10. Address clash scenario - select verified address
  11. Account area - review address clash warning
  12. Explore perceived pros and cons of legacy data with user

Objectives

  1. Is there any appetite for profile completion?
  2. Do users identify the benefits of a data rich profile, and are they willing to put the effort in?
  3. Do users understand the concept of legacy data pull?
  4. Are users willing to add legacy data, are there some things they would rather keep separate?
  5. Do users acknowledge the importance of upkeep of consistent and correct information, such as address; what role does the GOV.UK account play in this? (is our initial concept of indicating inconsistencies and clashes as expected)
  6. (less important) Identify any issues with the shortened version of PYI flow

Generally:

Do users articulate effort vs. reward across concepts of adding data, pulling legacy data, going through PYI.

We are more interested in the concepts at this stage, rather than a detailed evaluation of UI and content.

Hypotheses / assumptions

  • We believe that users will express willingness to add data to their profile if there is a clear reason for doing so
  • We believe that users will express willingness to prove their ID as part of the account set up because of the perceived benefits of verified ID enabling quicker transactions (this is derived from comments in previous rounds of user research)
  • We believe that users will express willingness to connect legacy data if there is a clear reason for doing so
  • We believe that users will appreciate the control and choice over which pieces information / legacy data to connect to the GOV.UK account

Research Dates
